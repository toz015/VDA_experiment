# VDA Experiment — Update Note 2026-05-17

This document describes the files added or revised in this update.
The goal is to (i) explain what each file does, (ii) state the
mathematical formulation it implements or evaluates, and (iii) give
the headline empirical numbers it produces, so that nothing in the
artefacts requires re-reading the code to understand.

All accuracies below are step-attribution accuracy on the Who&When
benchmark, with K = 3 discriminators
(D0 = Gemini-3-flash, D1 = Qwen3-Next-80B-A3B-Instruct, D2 = GPT-OSS-120B).
HC = Hand-Crafted (n = 55 traces with in-range ground truth),
AG = Algorithm-Generated (n = 126).


## 1. Mechanism implementations

These belong under `vcg/` in the repository.

### `vcg/allocation_huber.py` — squared-truncated (Huber) VCG kernel

Implements the multiplicative VCG with the **truncated squared** kernel

    kappa_c(x) = 1 − min(x^2, c^2),    x ∈ [−1, 1].

The value function and coordinate FOC retain the original form

    v_k(d, theta_hat^k) = ∏_t kappa_c(d_t − theta_hat^k_t),
    d_t = WeightedMean_{k ∈ A_t(d)}( theta_hat^k_t ;  w_k^t(d) ),

where the active set is A_t(d) = { k : |d_t − theta_hat^k_t| < c }.
Sanity-checked to recover the original VCG exactly at c = 1.0.

Empirical: c = 0.05 gives HC 10.9%, AG 32.5%. Improves on the original
VCG on AG but not significantly (p = 0.15); does not address the HC
failure mode (see L1-Huber below).

### `vcg/allocation_l1huber.py` — L1-Huber VCG kernel

Implements the multiplicative VCG with the **truncated absolute-value**
(L1-Huber) kernel

    kappa_c(x) = 1 − min(|x|, c),    x ∈ [−1, 1].

The value function is unchanged in form,

    v_k(d, theta_hat^k) = ∏_t kappa_c(d_t − theta_hat^k_t).

The mechanism-design content is in the coordinate FOC, which becomes a
**reputation-weighted median** over the active set rather than a
weighted mean:

    d_t  =  WeightedMedian_{k ∈ A_t(d)}( theta_hat^k_t ;  w_k^t(d) ),
    A_t(d) = { k : |d_t − theta_hat^k_t| < c },
    w_k^t(d) = ∏_{t' ≠ t} kappa_c(d_{t'} − theta_hat^k_{t'}).

The substitution mean → median is the entire change from the original
mechanism. Because kappa_c ≥ 1 − c > 0, the reputation never collapses
to zero and every discriminator retains a non-trivial voice at every
step.

Empirical: c = 0.05 gives HC 23.6% and AG 30.2%. HC improvement over
original VCG is +14.5pp (McNemar p = 0.022) and over Mean is +10.9pp
(p = 0.031). This is the main HC result of the update.


## 2. Diagnostic and ablation scripts

These belong under `scripts/`. They are independent and can be run in
any order. Each prints a self-contained report.

### `scripts/diagnostic_v1.py` and `scripts/diagnostic_v2.py`

Initial diagnostic scripts. `diagnostic_v1.py` reports the oracle
upper bound (HC 27.3%, AG 50.0% if you could pick the best k for each
trace), anchoring profile, saturation statistics, and per-discriminator
correlation. `diagnostic_v2.py` reports per-discriminator argmax-step
accuracy and per-method × epsilon grids for the original VCG. These
established the two failure modes diagnosed in Section 3 of the
research note.

### `scripts/run_ablation_huber.py`

Sweep over kernel and bandwidth. Includes the original VCG,
Mean (probability), Median (probability), the squared-Huber kernel,
and the L1-Huber kernel, all at multiple values of c in
{0.03, 0.05, 0.08, 0.10, 0.20, 0.50, 1.00}. Also includes VCG-Select
variants and oracle-over-methods rows for upper-bound reference.

Used to verify that c = 0.05 is not a singular peak; nearby c values
({0.03, 0.05, 0.08}) all give HC ≥ 18% with L1-Huber. The MAD-adaptive
bandwidth analysis is also implemented here: on HC, 100% of traces hit
the lower clip 0.05, so the data-adaptive selector reduces to fixed
c = 0.05.

### `scripts/explore_ag.py`

The per-regime breakdown reported in Table 3 of the research note.
Classifies each trace into one of three regimes by the agreement
structure of the K = 3 discriminators' argmax_t:

  Unanimous     — all three discriminators have the same argmax_t,
  2-of-3 agree  — exactly two have the same argmax_t,
  All differ    — all three argmax_t are distinct.

For each (method, regime) pair, prints conditional step accuracy. This
is the diagnostic that established the key per-regime findings:

  HC, All-differ regime (33/55 traces):
    L1-Huber 15.2%,  Median 9.1%,  Mean 3.0%,  original VCG 6.1%.

  AG, All-differ regime (44/126 traces):
    no within-trace mechanism exceeds 27.3%, while Gemini alone
    achieves 36.4% — the within-trace ceiling.

Also implements the selection and per-discriminator-normalised
aggregation methods explored during AG investigation (peak, entropy,
peak-separation, Gini, z-score mean, min-max mean, rank mean, plus
consensus-or-fallback hybrids).

### `scripts/bootstrap_ci.py`

Bootstrap 95% confidence intervals (B = 5,000 resamples, seed 42)
and McNemar exact paired tests for the methods reported in the main
results table.

Produces all p-values cited in Section 4.1 of the research note:

  HC, L1-Huber vs original VCG:        +14.5pp, p = 0.022 (n10/n01 = 9/1)
  HC, L1-Huber vs Mean(prob) eps=1e-6:  +10.9pp, p = 0.031 (n10/n01 = 6/0)
  HC, L1-Huber vs Median:                +3.6pp, p = 0.50  (n10/n01 = 2/0)
  AG, Confidence-Select vs original:     +7.9pp, p = 0.099 (n10/n01 = 20/10)
  AG, Confidence-Select vs Mean(0.10):   +0.8pp, p = 1.00  (n10/n01 = 11/10)


## 3. Unification attempts (Section 5 of the research note)

These scripts implement the two unification ideas — both fail, and
both failures are themselves discussed as findings in the note. Files
belong under `scripts/`.

### `scripts/cross_trace_prior.py` — cross-trace VCG-payment prior

Two-pass mechanism. Let Pi_k^{(j)} denote the L1-Huber VCG payment of
discriminator k on trace j, computed by the leave-one-out construction

    Pi_k^{(j)}  =  S_{−k}(d^*_{−k, j})  −  S_{−k}(d^*_j),

where S_{−k}(·) is the L1-Huber social welfare excluding k and d^*_j,
d^*_{−k, j} are the L1-Huber allocations on trace j with all K and
with k removed respectively.

Pass 1: compute (Pi_k^{(j)}) for every trace j ∈ [N], using three
parallel aggregation statistics (sum-payment, mean-payment,
mean-cross-step-reputation).

Pass 2: for each test trace i, form the **leave-one-out** prior

    pi_k^{(i)}  ∝  aggregate_{j ≠ i}  Pi_k^{(j)},

and use it either as a hard selection signal

    k^*  =  argmax_k  pi_k^{(i)},     hat_t^*  =  argmax_t  theta_hat^{k^*}_t,

or as a multiplicative augmentation of the within-trace reputation

    w_k^t   ←   pi_k^{(i)}  ·  ∏_{t' ≠ t} kappa_c(d_{t'} − theta_hat^k_{t'}),

and run the L1-Huber Gauss-Seidel allocation with the augmented
weight.

The leave-one-out construction prevents in-sample contamination.

Empirical (AG, n = 126, c = 0.05): all three aggregation statistics
nominate D1 (Qwen) as the top discriminator, with payment sums
(D0, D1, D2) = (4.89, 11.16, 3.10). But the actual argmax-step
accuracies are (38.1%, 27.0%, 19.0%) — the prior consistently nominates
the wrong discriminator. pi-select scores 27.0% on AG (exactly Qwen's
solo accuracy); pi-weighted L1-Huber scores 29–31%, indistinguishable
from the L1-Huber baseline of 30.2%.

The reason is structural: VCG payment measures consensus-shift, so it
rewards consensus-followers and penalises lone correct experts. This
is what the informal Proposition 3 in the note states.

### `scripts/unified_mechanism.py` — within-trace evidence prior

Avoids cross-trace contamination entirely. The prior pi_k depends
only on the current trace's reports, via one of three intrinsic
peakiness measures:

    pi_k^{peak}     =  max_t  theta_hat^k_t,
    pi_k^{peaksep}  =  theta_hat^{(1)}_{k,·}  −  theta_hat^{(2)}_{k,·},
    pi_k^{negent}   =  exp( −H_k ),    H_k = entropy( theta_hat^k_· / ∑_t · ).

The augmented reputation is

    w_k^t   =   (pi_k(theta_hat))^alpha   ·   ∏_{t' ≠ t} kappa_c(d_{t'} − theta_hat^k_{t'}),

where alpha is a sharpening exponent. alpha = 0 recovers L1-Huber;
alpha → ∞ approaches hard selection on pi_k.

The script also tests a gap-adjusted decision rule

    hat_t^*  =  argmax_t  ( d_t  +  beta · g_t ),

where g_t = top1 − top2 across discriminators k at step t is the
per-step cross-discriminator gap, and beta is a tunable weight.

Empirical (alpha scan across alpha ∈ {0, 0.5, 1, 2, 4, 8, 16, 64},
both subsets, all three priors):

  Best AG accuracy achieved by any setting: 33.3% (peak prior at alpha = 64),
  still below Confidence-Select's 35.7%.

  HC accuracy never strictly improves over L1-Huber baseline 23.6%
  for any alpha > 0.

  Gap-adjusted decision reduces accuracy on both subsets at every
  tested (alpha, beta).

The two best mechanisms (L1-Huber and Confidence-Select) are not
parametric neighbours within this family. This is the empirical basis
for the "aggregation–selection dichotomy" discussion in Section 5.3
of the research note.

## 5. Recommended ordering for re-running everything from scratch

For someone wishing to reproduce all numbers in the research note:

  1. python scripts/diagnostic_v1.py       (oracle bounds, saturation)
  2. python scripts/diagnostic_v2.py       (per-discriminator accuracy)
  3. python scripts/run_ablation_huber.py  (kernel + bandwidth sweep)
  4. python scripts/explore_ag.py          (per-regime breakdown)
  5. python scripts/bootstrap_ci.py        (CIs and McNemar p-values)
  6. python scripts/cross_trace_prior.py   (Section 5.1 negative result)
  7. python scripts/unified_mechanism.py --scan-alpha
                                           (Section 5.2 alpha scan)

Total runtime on the project data is on the order of a few minutes;
the longest individual script (cross_trace_prior.py) is under two
minutes on AG and proportionally longer on HC because of trace length.

All scripts use random seed 42 and B = 5,000 bootstrap resamples by
default. Numbers reported in the note should reproduce exactly.
