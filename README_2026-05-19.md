# VDA Experiment — Update Note 2026-05-19

This note describes the changes since the previous update,
`README_2026-05-17.md`. The two updates together cover the full theoretical
and empirical evolution of the mechanism design in this project.

## What changed since 2026-05-17

The 2026-05-17 update introduced the L1-Huber kernel as the proposed
mechanism, with two operating points (L1-Huber for HC, Confidence-Select
for AG) and reported `R = 0` (no OMD) numbers HC `23.6%`, AG `35.7%`.

The 2026-05-19 update generalises the mechanism to a one-parameter kernel
family and integrates everything into a single static mechanism with a
two-layer equilibrium guarantee. The empirical headline number on AG
improves to `36.5%`. The OMD calibration question is resolved by direct
experiment plus a structural explanation.

## Summary of the new mechanism

### Kernel family

The mechanism is now defined by a one-parameter family of multiplicative
VCG kernels

    kappa_{c, p}(x)  =  1 - min(|x|^p, c^p),   p >= 1.

The exponent p interpolates between the L1-Huber kernel (p = 1, weighted
median coordinate update) and the original squared kernel (p = 2, weighted
mean coordinate update). Intermediate p produces an M-estimator
coordinate update on a weighted p-th-quantile.

Coordinate FOC inside the active set A_t(d) = {k : |d_t - theta^k_t| < c}:

    sum_{k in A_t} w_k^t * |d_t - theta^k_t|^{p - 1} * sgn(d_t - theta^k_t) = 0.

For p = 1 the FOC reduces to a weighted median (closed form). For p = 2 it
reduces to a weighted mean (closed form). For p in (1, 2) the FOC is
solved by 1-D bisection per coordinate, exploiting strict monotonicity in d_t.

### Two operating points

A (p, c) scan over Who&When identifies two operating points:

  (p, c, eps) = (1.00, 0.05, 1e-6)    long-trace anchor; HC optimum (23.6%)
  (p, c, eps) = (1.75, 1.00, 0.10)    short-trace anchor; AG optimum (36.5%)

Both achieved at R = 0 (no OMD calibration).

### Trace-length-adaptive selector

A smooth sigmoid selector picks the operating point from the (observable)
trace length T:

    w(theta_hat)  =  sigmoid((log T - mu) / tau),    (mu, tau) = (log 10, 0.1)

with (p, c, eps) the convex combination of the two anchors weighted by w.
T was identified as the cleanest within-trace observable for regime
separation: best threshold-based error rate 3.6%, versus 16% for kurtosis
and 22% for saturation rate.

A combined-score extension log T + alpha * kurt(theta_hat) was scanned
over (alpha, mu, tau); it never improves on T-only on Who&When because
the empirical data supports a step-like routing rather than a smooth
continuum. The combined formulation is retained as a documented extension
for benchmarks with intermediate noise regimes.

### Two-layer equilibrium guarantee

The mechanism is single-shot, with no OMD calibration loop. Justification
is given by two separate equilibrium results, both static:

  Theorem 2 (DSIC for Power-VCG): For every kernel exponent p >= 1 and
  bandwidth c, the Clarke pivot payment

      Pi_k  =  S_{-k}(d*_{-k})  -  S_{-k}(d*),    S_{-k}(d) = sum_{j != k} v_j(d)

  yields truthful reporting as a dominant-strategy equilibrium of the
  discriminator game. The argument is kernel-agnostic, following the
  standard VCG externality structure.

  Proposition 3 (Gauss-Seidel Coordinate-Wise Nash): The converged output
  d* of the Gauss-Seidel solver is a coordinate-wise Nash equilibrium of
  the social-welfare game over allocations:

      d_t*  in  argmax_{d_t}  V^{(p, c)}(d_1*, ..., d_t, ..., d_T*),   for all t.

  Proof: V is continuous, bounded, and each coordinate update solves the
  active-set FOC at its unique stationary point (by FOC monotonicity in
  d_t). The implementation's monotonicity guard ensures non-decreasing V,
  giving monotone convergence to the joint coordinate-wise stationary
  point.

Together, the two equilibria describe a static two-layer guarantee:
under truthful reporting (Theorem 2), the Gauss-Seidel allocation
(Proposition 3) realises the equilibrium prediction. No outer iteration
on reports is required.

### OMD calibration diagnostic

The companion pipeline supports R > 0 OMD calibration. We empirically
tested three OMD objectives at the AG operating point (p = 1.75, c = 1,
eps = 0.10, R = 5):

  payment-ascent  (Tong's default):  AG 32.5%   (R=0 baseline: 36.5%)
  entropy-ascent  (concentration):    AG 23.0%
  margin-ascent   (top1 - top2 gap):  AG 33.3%

All three degrade. The mechanism is consensus drift: each gradient,
through different functionals, pushes the allocation toward higher
cross-discriminator consensus. On AG the mistake step is structurally
off-consensus (one or two discriminators identify a step the others miss),
so consensus drift pulls the prediction away from truth. The same
mechanism explains an earlier finding (cross-trace payment prior nominates
Qwen, not the more accurate Gemini).

The note proposes truth-aligned OMD calibration as future work,
specifically a non-consensus calibration signal that tracks agreement
with a minority discriminator.

## Files added or changed

### Mechanism implementations under `vcg/`

  allocation_power.py        Power-kernel Gauss-Seidel solver with FOC.
                              p = 1 uses exact weighted median; p = 2 uses
                              exact weighted mean; p in (1, 2) uses
                              bisection on the active-set FOC.
  payment_power.py            Clarke pivot payments for the power kernel.
  gradient_power.py           Finite-difference gradient of Pi_k for OMD.
  pipeline_power.py           End-to-end Phase 1b + 2 + 3 + 4 with selectable p.
  gradient_concentration.py   Inference-aligned OMD gradients (entropy,
                              margin).

### Scripts under `scripts/`

  eps_sweep.py                Sweep external clip values for sanity check
                              against Tong's R = 5, eps = 0.10 baseline
                              (confirmed AG 34.9%).
  scan_pc.py                  (p, c) grid scan over both subsets at both
                              clip levels.
  regime_stats.py             Computes seven candidate within-trace
                              statistics and reports best threshold-based
                              error rate for HC vs AG separation. T wins
                              at 3.6% error.
  kurtosis_diagnose.py        Detailed kurtosis distribution per subset
                              plus T-kurtosis correlation analysis.
  adaptive_kernel.py          Hard trace-length threshold selector (T > 10
                              -> long anchor).
  adaptive_kernel_smooth.py   Smooth sigmoid selector over a single
                              statistic (T, kurtosis, saturation rate,
                              etc).
  adaptive_kernel_combined.py Smooth selector over the combined score
                              log T + alpha * kurtosis.
  run_power.py                Power-VCG pipeline runner, mirrors run_vcg.py
                              and run_l1huber.py.
  run_power_v2.py             Power-VCG runner with --omd-objective
                              in {payment, entropy, margin}.
  omd_diagnose.py             Single-trace OMD dynamics inspector
                              (gradient signs, payment magnitudes,
                              theta drift per round).
  omd_find_changers.py        Compares R = 0 vs R = 5 on all AG traces,
                              identifies the hit-to-miss / miss-to-hit
                              traces that explain the OMD accuracy drop.

## How to reproduce the headline numbers

From repo root:

    # Sanity check Tong's baseline (AG 34.9%)
    python scripts/run_vcg.py --subset Algorithm-Generated --R 5 --report-eps 0.10 --tag verify

    # (p, c) scan
    python scripts/scan_pc.py --report-eps 1e-6
    python scripts/scan_pc.py --report-eps 0.10

    # Separability of candidate routing statistics
    python scripts/regime_stats.py

    # Trace-length adaptive selector (canonical mechanism)
    python scripts/adaptive_kernel_smooth.py --mode T-only --scan

    # Smooth selector with kurtosis extension (combined score)
    python scripts/adaptive_kernel_combined.py --scan

    # OMD diagnostic at AG operating point
    python scripts/run_power_v2.py --subset Algorithm-Generated --p 1.75 --c 1.0 --report-eps 0.10 --R 5 --omd-objective payment --tag R5_payment
    python scripts/run_power_v2.py --subset Algorithm-Generated --p 1.75 --c 1.0 --report-eps 0.10 --R 5 --omd-objective entropy --tag R5_entropy
    python scripts/run_power_v2.py --subset Algorithm-Generated --p 1.75 --c 1.0 --report-eps 0.10 --R 5 --omd-objective margin --tag R5_margin

    # Per-trace hit/miss flip analysis under OMD
    python scripts/omd_find_changers.py

Total runtime is a few minutes for everything except the OMD experiments
on AG (each `--R 5` run is roughly 20 minutes on AG; correspondingly
longer on HC because of larger T).

Random seed 42 throughout. All numbers in the v3 research note are
deterministic.
